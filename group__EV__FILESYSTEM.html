◆ ev_file_close()

void ev_file_close ( ev_file_t * file,
ev_file_close_cb cb )

Close a file handle.

If the file is open in synchronous mode (the callback in ev_file_open() is set to NULL), then this is a synchronous call. In this case cb must be NULL.

If the file is open in asynchronous mode, this call is also asynchronous, you must wait for cb to actually called to release the resource.

Parameters
[in]fileFile handle
[in]cbClose callback. Must set to NULL if file open in synchronous mode.

◆ ev_file_mmap()

int ev_file_mmap ( ev_file_map_t * view,
ev_file_t * file,
uint64_t offset,
size_t size,
int flags )

Maps a view of a file mapping into the address space of a calling process.

Parameters
[out]viewThe mapped object.
[in]fileThe file to map. The file is safe to close after this call.
[in]offsetThe offset where the view is to begin. It must be a multiple of the value of ev_os_mmap_offset_granularity(). You can use EV_ALIGN_SIZE() to align the offset to requirements. The offset can larger than the file size, in this case the gapping data is undefined.
[in]sizeThe maximum size of the file mapping object.
  • If offset is in range of file, set to 0 to use the range from offset to the end of the file.
  • If offset is not less than file size, it must larger than 0.
[in]flagsMap flags. Can be one or more of the following attributes:
  • EV_FS_S_IRUSR: Pages may be read.
  • EV_FS_S_IWUSR: Pages may be written.
  • EV_FS_S_IXUSR: Pages may be executed.
  • EV_FS_S_IRWXU: Combine of read, write and execute. Please note that this is a best effort attempt, which means you may get extra permissions than declaration. For example, in win32 if you declare EV_FS_S_IXUSR only, you will also get read access.
Returns
ev_errno_t

int ev_file_open ( ev_loop_t * loop,
ev_file_t * file,
ev_fs_req_t * req,
const char * path,
int flags,
int mode,
ev_file_cb cb )

Equivalent to open(2).

The full list of flags are:

  • EV_FS_O_APPEND
  • EV_FS_O_CREAT
  • EV_FS_O_DSYNC
  • EV_FS_O_EXCL
  • EV_FS_O_SYNC
  • EV_FS_O_TRUNC
  • EV_FS_O_RDONLY
  • EV_FS_O_WRONLY
  • EV_FS_O_RDWR

The full list of mode are:

  • EV_FS_S_IRUSR
  • EV_FS_S_IWUSR
  • EV_FS_S_IXUSR
  • EV_FS_S_IRWXU
Note
File always open in binary mode.
Parameters
[in]loopEvent loop. Must set to NULL if cb is NULL.
[out]fileFile handle.
[in]reqFile token. Must set to NULL if cb is NULL.
[in]pathFile path.
[in]flagsOpen flags.
[in]modeOpen mode. Only applies to future accesses of the newly created file. Ignored when flags does not contains EV_FS_O_CREAT.
[in]cbOpen result callback. If set to NULL, the file is open in synchronous mode, so loop, req must also be NULL.
Returns
ev_errno_t

Range Means

< 0 ev_errno_t
Parameters
